# Manual Smoke Checklist

## 0. 启动准备

1. 双击 `start_study_desk.bat`。
2. 预期浏览器自动打开 `http://127.0.0.1:4173/index.html`。
3. 如果本机没有 Python 3，预期脚本明确提示，而不是静默失败。
4. 顶部状态条应显示本地记录已就绪；默认不出现 LLM 可用提示。

## 1. 今日页布局稳定性

在桌面浏览器下分别验证 100%、110%、125%、150% 缩放。

1. 打开 `今日画布`。
2. 预期页面主体仍然是：
   - 左侧大画布
   - 右侧今日时间安排卡
3. 预期时间片不会掉到底部变成普通块。
4. 收起时间片后，右侧保留窄入口；再次展开后恢复完整时间片。
5. 点击任务后，大右抽屉浮出，不挤压左侧画布与右侧时间片。

## 2. 大右抽屉

1. 在今日画布点击已有任务。
2. 预期打开统一的大右抽屉，而不是在页面下方展开表单。
3. 点击 `新建学习卡片`。
4. 预期创建新任务并直接打开同一个大右抽屉。
5. 给任务添加较长的 notes 与较长的任务延续区内容。
6. 预期：
   - 抽屉头部固定
   - 内容区内部出现滚动
   - 页面主布局不被撑开

## 3. 今日时间安排卡

1. 在今日页确认存在三个固定时间片：
   - 上午
   - 下午
   - 晚上
2. 没有任务时，时间片显示空态提示。
3. 从左侧画布拖一个任务到 `上午`。
4. 预期：
   - 右侧出现对应任务引用
   - 左侧原任务卡片保持原样，不自动折叠
   - 左侧卡片新增轻标记，例如 `已安排：上午`
5. 再把同一个任务加入 `晚上`。
6. 预期左侧标记变成 `已安排：上午 / 晚上`。
7. 尝试再次加入同一时间片。
8. 预期同一时间片内不会重复。
9. 在时间片内使用 `上移 / 下移`。
10. 预期顺序变化并在刷新后保持。
11. 从时间片移除任务。
12. 预期：
   - 只删除时间片引用
   - 左侧原任务卡片仍存在
   - 卡片其他信息不受影响

## 4. 卡片展示语义

1. 拖入时间片前后，对比左侧卡片展示。
2. 预期卡片仍保留：
   - 状态
   - notes 数量
   - 正文提示
   - 大小与位置
3. 已完成任务加入时间片后，右侧引用应显示 `✓` 且文字变灰。
4. 高重要且高紧急任务在右侧引用中应显示 `★`。
5. 从过去安排来的任务在右侧引用中应显示 `↪` 或 `from yyyy/mm/dd`。

## 5. 安排到指定日期

1. 选中今日任务，点击 `安排到日期`。
2. 验证以下情况：
   - 只带标题
   - 带部分 notes
   - 带任务延续区字段
   - 带一句日历
3. 确认创建后：
   - 目标日期出现新任务
   - 新任务写入 `scheduledDate`
   - 新任务写入 `arrangedFrom`
   - 新任务写入 `isArrangedTask`
4. 返回今日页或目标日期后，确认旧任务和新任务都正常可编辑。

## 6. 月历模式回归

1. 打开 `月历模式`。
2. 预期月视图正常渲染，不受今日页改造影响。
3. 检查每天任务标记：
   - `↪`
   - `★`
   - `✓`
   - `+n`
4. 点击日期。
5. 预期右侧日期辅助面板打开。
6. 点击月历中的任务。
7. 预期打开同一个大右抽屉。
8. 对一个带很多 notes 的任务重复验证。
9. 预期抽屉内部滚动稳定，不需要缩放页面才能看完整内容。

## 7. 历史记录回归

1. 打开 `历史记录`。
2. 切换到有数据的日期。
3. 预期可查看：
   - 当天总结
   - 任务 notes
   - 任务延续区
4. 默认无 LLM 模式下，历史页不应再展示任务内对话区块。

## 8. LLM 关闭验证

1. 今日页任务抽屉中不应出现：
   - LLM 对话列表
   - 聊天输入框
   - “发送给 LLM”
2. 今日总结弹窗中不应出现：
   - “生成 / 刷新草稿”
3. 安排到日期弹窗中不应出现：
   - “带上 LLM 对话摘要”
   - “下一步计划”类 LLM 衍生选项
4. 顶部状态条不应显示 “LLM 已就绪”。

## 9. 旧数据兼容

1. 用已有 `data/tasks.json`、`data/notes.json`、`data/records.json` 启动项目。
2. 预期旧任务仍可正常显示、编辑、归档、安排到日期。
3. 如果 `data/time_blocks.json` 中引用了不存在的任务：
   - 渲染时应自动过滤
   - 后续保存时应清理孤儿引用

## 10. 启动方式回归

1. 关闭所有 Study Desk 窗口后，再次双击 `start_study_desk.bat`。
2. 预期：
   - 服务可再次启动
   - 浏览器可再次自动打开
   - 默认仍是无 LLM 模式
